推送相关

发表于 2021-01-28 更新于 2021-08-21 分类于 iOS ， 相关
本文字数： 3.3k 阅读时长 ≈ 3 分钟

iOS 推送分为本地推送、在线推送、离线推送。

  • 本地推送就是本地可以通过固定时间以及地点等条件触发推送操作。
  • 在线推送就是 APP 在前台利用长链接等形式实现推送，如 QQ、微信在前台时收到的消息；
  • 离线推送是指通过 APNs 服务器发送通知到 APP。

离线推送

推送流程基本如下：

  1. APP 申请推送权限
  2. APP 申请获得 deviceToken
  3. APNs 服务器验证成功后，将 deviceToken 返回至 APP 端
  4. APP 端将 deviceToken 存储至服务器，服务器带着证书以及 deviceToken 发送到 APNs 服务器
  5. APNs 服务器下发推送消息到 APP 端

跨域原理及解决

相关原理

跨域问题是在浏览器中出现的情况，主要原因是因为浏览器本身的同源策略机制的限制，同源策略（Same Origin Policy，简称SOP问题）是一种安全策略；

URL= 协议 + 域名 + 端口 + 协议；如果两个 URL 的协议、域名和端口相同，则它们同源，如果不相同，则不同源，则就会引起跨域；

浏览器会限制脚本中发起的跨域请求。比如，使用 XMLHttpRequest 对象和 Fetch 发起 HTTP 请求就必须遵守同源策略。不允许跨域访问并非是限制浏览器发起请求，而是返回的结果被浏览器给拦截了，接收不到；最好的例子是 CSRF 跨站攻击原理，无论是否跨域，服务器都接收到了请求。

iOS 多线程 -GCD

发表于 2021-01-26 更新于 2021-12-16 分类于 iOS ， 多线程
本文字数： 9.6k 阅读时长 ≈ 9 分钟

前言

Hi Coder，我是 CoderStar！

今天给大家带来多线程系列的第二篇文章 — GCD，其大概率是我们在使用多线程时最常用的方式了。

GCD 全称是 Grand Central Dispatch，翻译过来就是大规模中央调度。根据官方文档，它的作用是：通过向系统管理的调度队列中提交任务，在多核硬件上同时执行代码。它提供了一套机制，让你可以充分利用硬件的多核性能，并且让你不用再调用那些繁琐的底层线程 API，编写易于理解和修改的代码。

对开发者而言，面对的不再是上一篇文章iOS多线程-Thread所描述的线程，CGD将线程概念模糊掉，开发者转而面对的是更上层的队列和任务，不再需要考虑线程的周期以及调度等等，这些交由GCD内部处理就好。
